**Shebang + safety pragmas** (top of file): `#!/usr/bin/env bash`, then `set -euo pipefail`, then `IFS=$'\n\t'`.

**Configuration block** (after the shebang, before the checks):

- `IN_SCOPE_DOCS` is a bash array listing the six in-scope document paths relative to `REPO_ROOT`: `docs/phase-2/cardano-realism-audit.md`, `docs/phase-2/validity-threats.md`, `docs/phase-2/realism-risks-register.md`, `docs/phase-2/coverage-check.md`, `docs/phase-2/methodology-overview.md`, `docs/phase-2/cip-author-summary.md`. - `SUITES_DIR` is `sim-rs/parameters/phase-2-sweep/suites`.
- `GOLDENS_DIR` is `sim-rs/parameters/phase-2-sweep/suites/.goldens`.
- `EXIT_CODE=0` (mutated to 1 on any check failure).

**Check (i): RSK-NN / CLM-NN / EXP-NN dead-reference scan.** Implementation:
- Extract canonical RSK-NN set from `docs/phase-2/realism-risks-register.md` via `grep -oE '^## RSK-[a-z0-9-]+' | sed 's/^## //' | sort -u`. The metasyntax token `RSK-NN` itself is intentionally not in the canonical set; exclude any token matching exactly `RSK-NN` from the scan when comparing references.
- Extract canonical CLM-NN set from `docs/phase-2/coverage-check.md` via `grep -oE '^\| CLM-[0-9]+' | sed 's/^| //' | sort -u`. (The leading `| ` anchors on the table-row pattern; rows like `| CLM-01 | ...` define the canonical IDs.) Exclude `CLM-NN` metasyntax.
- Extract canonical EXP-NN set from `docs/phase-2/realism-risks-register.md` via `grep -oE 'EXP-[a-z0-9-]+' | sort -u`. The register is the EXP-NN definition site per the existing append-only convention. Exclude `EXP-NN`, `EXP-<slug>`, and `EXP-slug` metasyntax tokens (these are documentation placeholders, not real identifiers).
- For each of the six in-scope documents that exist on disk: extract every `RSK-[a-z0-9-]+`, `CLM-[0-9]+`, `EXP-[a-z0-9-]+` token; for each token, check whether the token is in the corresponding canonical set; collect dead references (tokens not in the canonical set, excluding the listed metasyntax tokens).
- Output JSON-ish per-document tallies and a global PASS/FAIL: PASS iff total dead references across the six documents is 0.

**Check (ii): `backing-job` path resolution.** Implementation:
- Parse `docs/phase-2/coverage-check.md` for the `backing-suite` and `backing-job` columns. The shape of each CLM row is a markdown table; the `backing-suite` cell is at position 6 (1-indexed; columns: id, claim, menu-option, status, confidence-method, backing-suite, backing-job, seeds-cited, ...). Use `awk -F '|' '/^\| CLM-/ { gsub(/^[ \t]+|[ \t]+$/, "", $7); gsub(/^[ \t]+|[ \t]+$/, "", $8); print $7"|"$8 }'` to extract `(backing-suite, backing-job)` pairs. Skip rows where either cell is `—` (no backing-suite or no backing-job — these are structural / by-construction claims).
- For each `(backing-suite, backing-job)` pair: resolve `backing-suite` to its path-relative-to-REPO_ROOT form, confirm the suite YAML exists, then run `yq '.jobs[].name'` against the suite YAML and confirm the `backing-job` slug is one of the returned job names.
- Tally: total pairs checked, pairs resolved successfully, pairs that failed resolution. PASS iff failed = 0.

**Check (iii): `golden-sha256` cross-check.** Implementation:
- Parse `docs/phase-2/coverage-check.md` for the `golden-sha256` column (column position 10 in the row layout, 1-indexed; columns: id, claim, menu-option, status, confidence-method, backing-suite, backing-job, seeds-cited, golden-sha256, ...). Use a similar `awk -F '|'` extraction. Cells may carry annotations in parentheses after the hash (e.g. `92701c73944e`, `af6adc822c3a (legacy phase-2 seed=1; Phase 3 seeds-cited 20 distinct)`); extract the leading 12 hex chars only (`grep -oE '^[a-f0-9]{12}'` post-trim).
- Skip rows where the cell is `—` (structural / by-construction claims).
- For each `(backing-suite, golden-sha256-truncated-12)` pair: locate `.goldens/<suite-basename>.sha256` (derived from the suite YAML path by stripping the suite YAML extension), then `grep -F "$truncated_hash"` against the goldens file. PASS iff every truncated hash matches at least one full hash in the corresponding goldens file.
- Special case: if a row's `backing-suite` is `sim-rs/parameters/phase-2-sweep/suites/phase-3-canonical-variance.yaml` or any other suite not in the seven goldens-pinned set (the seven `.sha256` files in `.goldens/`), the row is exempt from this check — annotate as `exempt (non-pinned suite)` in the report rather than counting as a failure. Simplification accepted: if the row's `backing-suite` is a `phase-3-*` suite, look up the legacy pinned suite by scanning sibling CLM rows for the same hash and match against the pinned-suite goldens file. If no match in any pinned-suite goldens file, the row counts as a failure.
- Tally: total hashes checked, hashes matched, hashes exempt (non-pinned), hashes failed. PASS iff failed = 0.

**Check (iv): markdown link resolution.** Implementation:
- For each of the six in-scope documents that exist on disk: extract every markdown link of the form `[text](path)` via `grep -oE '\[[^]]+\]\([^)]+\)'`, then extract the path component via `sed -E 's/.*\(([^)]+)\)/\1/'`. Strip `#anchor` suffixes via `sed 's/#.*$//'`. Skip protocol-prefixed URLs (http://, https://, mailto:).
- For each relative path: resolve against the document's directory (e.g. for `docs/phase-2/cardano-realism-audit.md`, resolve `../../sim-rs/parameters/...` against `docs/phase-2/`) and confirm the resolved file exists on disk.
- Also extract backtick-wrapped path references via `grep -oE '\`[^\`]*\.(md|sh|yaml|yml|toml|rs|py|json|txt)\`'` (paths to files with known extensions). Strip backticks. Resolve relative to the document's directory if relative, else against REPO_ROOT if starting with `sim-rs/`, `docs/`, `.planning/`, or `parameters/`. Confirm each resolves to an extant file. Skip metasyntax tokens like `\`path/file.ext\`` if the contents contain placeholder markers like `<...>` or `NN`.
- Tally: total links checked, links resolved, links broken. PASS iff broken = 0.
