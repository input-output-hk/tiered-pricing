# [Experimentation](https://github.com/input-output-hk/tiered-pricing/issues/54)

Built a new, more focused, network-abstracted simulator for quicker and more direct testing of hypotheses. [PR here](https://github.com/input-output-hk/tiered-pricing/pull/75/changes).

Functionality so far includes:

- Configurable candidate designs covering the axes under consideration: one or two lanes; no dynamic pricing (control), standard-only dynamic, priority-only dynamic, or both lanes dynamic; priority space reservation in RBs or no reservation; FIFO / priority-first / capped-standard block selection; and fixed-fee, EIP-1559 (posted max fee with refund), or honour-submission-quote fee semantics
- Per-lane EIP-1559-style price controllers with configurable target utilisation, maximum change rate, update signal (capacity-weighted window or priority-reservation utilisation), and price floors
- Linear-Leios block production: EB certification/inclusion probability derived from Praos timing, mempool admission/staleness/eviction rules, and blocks as the native unit for latency
- Configurable rational actors (honest, patient, and impatient behaviours so far) that choose lane and posted fee based on quoted price, transaction value, urgency, and expected per-lane latency
- Transaction generation from configurable curves for tx size, Plutus script size, ex-units, and value, with fees computed as a lane coefficient over the full Cardano min fee
- Load shaping via constant or bursty arrival processes, with bursts able to scale transaction value and urgency
- The metrics from the experiment design: inclusion, retained/lost value, and inclusion latency by urgency class, lane, and urgency×lane; price shock; revenue (fees + refunds); aggregate throughput / EB utilisation; price convergence/oscillation; and invariant breach detection. (A fairness metric was tried and removed as not useful at this stage.)
- JSON configuration of both the simulation parameters and the pricing design
- A full per-run event trace (`events.jsonl`)

Also built an interactive dashboard (`abstract-sim-viz`, in the same PR) over the event traces: price per lane with convergence bands, price shock, latency by urgency class and lane, RB-content and EB-fullness timelines, inclusion/fate and retained-vs-lost value breakdowns, and a linked submission→inclusion flow panel, with brush-to-zoom synced across panels.

Next:
* Finish the remaining modelling from the experiment design (e.g. transaction dependency chains, adversarial/MEV actor behaviours)
* Run the candidate designs and compare results across the metric set

# [Prototype](https://github.com/input-output-hk/tiered-pricing/issues/55)

Nicolas returned from holiday on June 6th and finished getting acquainted with the project: linear-Leios ([CIP-0164](https://github.com/cardano-foundation/CIPs/tree/master/CIP-0164)), the current mempool design, the [phase-2 mechanism design doc](https://github.com/input-output-hk/tiered-pricing/blob/main/docs/phase-2/mechanism-design.md), [Polina's spec work](https://github.com/IntersectMBO/formal-ledger-specifications/compare/polina/dynamic), and Will's simulations — enough to start the prototype with a bottom-up approach: ledger → mempool → eventually a private-network simulation of the implementation.

Started implementing the ledger rules from Polina's spec on the `leios-prototype` branches. The first objective is to surface challenges in the current proposition vs the current Leios implementation.
